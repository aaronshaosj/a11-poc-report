# Claude Code Session 启动提示词

## 启动命令

在终端中进入 `a11-poc-report` 仓库目录后，使用以下提示词启动 Claude Code session：

---

### 提示词（复制以下内容）

```
请仔细阅读本项目根目录下的 CLAUDE.md 和 A11-POC-REPORT-DEV-PROMPT.md 两份文档。这两份文档描述了 C-ROS 智能调度系统中 A11 POC 报告生成器模块的完整开发需求。

你的任务是：

1. 深入理解文档背后的真实业务需求——这是一个物流调度优化系统的 POC 验证报告工具，核心价值在于通过多维度量化指标的可视化对比，向客户证明算法调度相比人工调度的优势。

2. 独立、客观地分析文档中提出的技术方案。如果你认为某些设计决策可以优化（例如图表类型选择、组件拆分方式、数据结构设计等），请在开发前先简要说明你的改进建议和理由，然后按照你认为最优的方案执行。

3. 基于 React 19 + TypeScript + Tailwind CSS 4 + ECharts 技术栈，完成完整的前端开发工作。项目使用 pnpm 管理依赖，Vite 构建。请先初始化项目脚手架，安装必要依赖，然后逐步实现所有页面和组件。

4. 确保以下核心功能完整实现：
   - 报告列表页（4 条 Mock 记录，覆盖已完成/生成中/失败三种状态）
   - 生成配置弹窗（三步引导：选批次 → 选策略 → 确认生成）
   - 报告内容页（16 张 ECharts 交互式图表 + 5 项 KPI 概览 + AI 解读文字 + 浮动导航）
   - 深色星空主题（Glassmorphism Deep Space），与 C-ROS Workbench 风格一致

5. Mock 数据必须合理且有区分度——不同策略之间的指标应有可辨别的差异，能体现各策略的优化倾向。

6. 开发完成后，做好充分的测试。在 tests/TEST-REPORT.md 中记录完整的测试报告，包括功能验证清单（逐项 PASS/FAIL）、发现的问题及修复状态、整体质量评估。

7. 将最终的实施方案说明（你做了哪些设计决策、与原方案的差异、技术亮点）和测试报告一并提交到 GitHub。

请开始工作。
```

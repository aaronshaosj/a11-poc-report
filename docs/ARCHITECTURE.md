# A11 POC 报告生成器 — 架构设计与功能实现方案

> **版本**: v1.0 | **日期**: 2026-03-06 | **模块编号**: A11 | **所属系统**: C-ROS Agentic Workbench

---

## 一、设计概述

A11 POC 报告生成器是 C-ROS Agentic Workbench 中的核心分析模块，旨在将人工调度与算法调度的仿真结果进行多维度、可交互的量化对比分析。本方案在现有 POC 报告（静态 PNG 图片 + Excel 表格）的基础上进行全面升级，实现以下四项核心迭代目标：

| 迭代目标 | 原方案 | 新方案 |
|---------|--------|--------|
| 图表呈现 | 独立 PNG 静态图片 | ECharts 交互式图表，统一为单个 HTML 文件 |
| 数据解读 | 无 | 每张图表配备 LLM 生成的凝练解读文字 |
| 策略对比 | 仅人工 vs 算法（2组） | 支持 n 组策略同时对比，动态扩展 |
| 生成流程 | 手动触发、无管理界面 | 完整的 UI 流程：批次选择 → 策略选择 → 异步生成 → 记录管理 |

## 三、UI 设计方案

### 3.1 页面结构总览

本方案包含三个核心页面/视图，均遵循 C-ROS Workbench 的深色星空主题（Dark Space Theme），保持与现有系统的视觉一致性。

| 页面 | 功能 | 入口 |
|------|------|------|
| 报告列表页 | 管理所有 POC 报告记录 | 左侧导航 → POC 报告 |
| 生成配置弹窗 | 三步引导式配置 | 列表页右上角「生成 POC 报告」按钮 |
| 报告内容页 | 查看完整的交互式报告 | 点击列表中的报告卡片 |

### 3.2 报告列表页

报告列表页采用**卡片式瀑布流布局**，每张卡片代表一次报告生成记录。卡片设计遵循信息密度适中的原则，在一屏内展示关键元数据，同时通过状态标签和颜色编码快速传达报告状态。

**卡片信息结构**：

| 信息项 | 说明 | 示例 |
|--------|------|------|
| 报告标题 | 自动编号 | POC 对比报告 #3 |
| 状态标签 | 已完成 / 生成中 / 生成失败 | 绿色「已完成」 |
| 订单批次 | 所选批次名称列表 | 17-1, 苏南17, 苏南18 |
| 对比策略数 | 策略组数（含人工经验） | 3 组（含人工经验策略） |
| 策略标签 | 彩色标签展示各策略名称 | 人工经验策略(橙) / 均衡优化 v2.1(蓝) |
| 图表/KPI 统计 | 仅已完成状态显示 | 包含 18 张交互式图表 · 5 项 KPI 概览 |
| 时间与耗时 | 生成时间和耗时 | 2026-03-06 14:32 · 耗时 2 分 18 秒 |
| 下载按钮 | HTML 和 Excel 两种格式 | 下载 HTML / 下载 Excel |

**状态设计**：

| 状态 | 颜色 | 附加信息 |
|------|------|---------|
| 已完成 | 绿色 (#10b981) | 显示图表数、KPI 数、下载按钮 |
| 生成中 | 蓝色 (#3b82f6) | 显示预计剩余时间、进度条 |
| 生成失败 | 红色 (#ef4444) | 显示错误原因、「重新生成」按钮 |

### 3.3 生成配置弹窗（三步引导）

生成流程采用 **Stepper 三步引导式弹窗**，顶部显示步骤指示器（① 选择批次 → ② 选择策略 → ③ 确认生成），每一步的界面设计如下：

**Step 1 — 选择订单批次**

系统自动从 C-ROS 平台加载当前项目下的所有订单批次数据。每个批次行显示批次名称、订单数量、站点数量和日期。提供「全选 / 取消全选」快捷操作，右上角实时显示已选择数量。用户至少需要选择 1 个批次才能进入下一步。

**Step 2 — 选择对比策略**

系统通过 API 获取当前项目中已配置的算法策略列表，并在列表顶部固定显示**「人工经验策略」**。人工经验策略使用橙色背景高亮并标注「基准」标签，默认选中且不可取消。算法策略使用蓝色「算法」标签，并在右侧显示迭代轮数和节降百分比等辅助信息。用户至少需要选择 1 个策略（含人工经验策略）。

**Step 3 — 确认生成配置**

以结构化摘要的形式展示用户的选择：订单批次列表、对比策略列表、预估耗时。底部显示提示信息，告知用户报告生成为异步过程，可以安全离开页面。点击「开始生成」后，弹窗切换为全屏 Loading 状态，显示旋转动画和进度文字，同时提供「后台运行」按钮允许用户返回列表页。

### 3.4 报告内容页

报告内容页是本方案的核心交付物，采用**单页长文档布局**，所有图表和解读按照指标分类体系（Type A / B / C）有序排列。

## 四、图表设计方案

### 4.1 图表分类体系

图表设计严格遵循 TuningAgent 定义的三类量化指标体系，确保与 C-ROS 仿真引擎的数据输出完全对齐。

| 分类 | 指标类型 | 对比逻辑 | 图表数量 |
|------|---------|---------|---------|
| Type A | 边界类指标 | 极值 / 分位数对齐 | 6 张 |
| Type B | 形态类指标 | 分布形状对齐 | 6 张 |
| Type C | 综合效能 | 多维度综合评价 | 4 张 |
| KPI | 核心概览 | 数值卡片 | 5 项 |

### 4.2 完整图表清单

**核心 KPI 概览（数值卡片）**

| KPI | 数据来源 | 展示方式 |
|-----|---------|---------|
| 总车次数 | 全批次汇总 | 大字数值 + 人工/算法对比差值 |
| 总里程 (km) | 全批次汇总 | 大字数值 + 百分比变化 |
| 总工作时长 (h) | 全批次汇总 | 大字数值 + 百分比变化 |
| 平均满载率 | 全批次加权平均 | 大字数值 + 百分比变化 |
| 里程节降率 | 算法最优 vs 人工 | 大字数值 + 目标区间标注 |

**Type A — 边界类指标（6 张图表）**

| 序号 | 图表名称 | 图表类型 | 数据维度 | 对比逻辑 |
|------|---------|---------|---------|---------|
| A1 | 各批次车次数对比 | 分组柱状图 | 按订单批次 × 策略 | 各批次的车次数横向对比 |
| A2 | 各批次平均满载率对比 | 折线图 | 按订单批次 × 策略 | 满载率趋势与差异 |
| A3 | 单车最大里程分布 | 箱线图 | 按策略分组 | 极值、中位数、四分位数对齐 |
| A4 | 单车最大作业时长 | 箱线图 | 按策略分组 | 时长分布的离散度对比 |
| A5 | 最大站点间隔对比 | 分组柱状图 | 按订单批次 × 策略 | 站点间距的极值控制能力 |
| A6 | 装载量上限利用率 | 雷达图 | 按车型（重量/体积/托盘/件数） | 多维装载约束的利用效率 |

**Type B — 形态类指标（6 张图表）**

| 序号 | 图表名称 | 图表类型 | 数据维度 | 对比逻辑 |
|------|---------|---------|---------|---------|
| B1 | 车次订单数分布 | 箱线图（含散点） | 按策略分组 | 每车订单数的分布形态 |
| B2 | 车次行驶距离分布 | 密度曲线图 | 按策略叠加 | 距离分布的峰值与偏态 |
| B3 | 线路跨度分布 | 直方图 | 按策略分组 | 外接圆直径的频率分布 |
| B4 | 点间距分布 | 累积分布函数(CDF)曲线 | 按策略叠加 | 站点间距的累积概率分布 |
| B5 | 跨区数量分布 | 堆叠柱状图 | 按跨区数 × 策略 | 跨区车次的占比结构 |
| B6 | 聚类离散度 (Top-K) | 散点图 | 按策略着色 | 最近邻距离的空间聚集程度 |

**Type C — 综合效能（4 张图表）**

| 序号 | 图表名称 | 图表类型 | 数据维度 | 对比逻辑 |
|------|---------|---------|---------|---------|
| C1 | 多维效能雷达图 | 雷达图 | 6 维归一化指标 | 策略间的综合能力对比 |
| C2 | 批次维度节降率趋势 | 折线图 | 按订单批次 × 策略 | 节降率随批次的变化趋势 |
| C3 | 工作时长 vs 行驶距离 | 散点矩阵图 | 按车次 × 策略着色 | 时长-距离的相关性与效率 |
| C4 | （预留）综合评分排行 | 水平柱状图 | 按策略 | 加权综合评分排名 |

### 4.3 图表类型多元化策略

为避免视觉单调，本方案在 16 张图表中使用了 **10 种不同的图表类型**，确保信息表达的多样性与准确性：

| 图表类型 | 使用场景 | 数量 |
|---------|---------|------|
| 分组柱状图 | 离散分类对比 | 2 |
| 折线图 | 趋势与连续变化 | 2 |
| 箱线图 | 分布统计（极值/四分位） | 3 |
| 雷达图 | 多维度综合评价 | 2 |
| 密度曲线图 | 连续分布形态 | 1 |
| 直方图 | 频率分布 | 1 |
| CDF 曲线 | 累积概率分布 | 1 |
| 堆叠柱状图 | 占比结构 | 1 |
| 散点图 | 相关性与聚集度 | 2 |
| 数值卡片 | KPI 概览 | 5 项 |

### 4.4 多策略扩展设计

图表的颜色方案采用**可扩展的策略调色板**，支持最多 8 组策略的同时对比：

| 策略位置 | 角色 | 颜色 | 色值 |
|---------|------|------|------|
| 第 1 位 | 人工经验策略（固定） | 琥珀橙 | #f59e0b |
| 第 2 位 | 算法策略 1 | 钢蓝 | #3b82f6 |
| 第 3 位 | 算法策略 2 | 翡翠绿 | #10b981 |
| 第 4 位 | 算法策略 3 | 紫罗兰 | #8b5cf6 |
| 第 5 位 | 算法策略 4 | 玫瑰红 | #f43f5e |
| 第 6 位 | 算法策略 5 | 青色 | #06b6d4 |
| 第 7 位 | 算法策略 6 | 粉色 | #ec4899 |
| 第 8 位 | 算法策略 7 | 石灰绿 | #84cc16 |

## 五、LLM 解读生成方案

### 5.1 解读定位

每张图表下方配备一段 **AI 生成的解读文字**，长度控制在 80-150 字之间，要求凝练、专业、有洞察力。解读文字应聚焦于数据差异的业务含义，而非简单复述数据本身。

### 5.2 Prompt 设计

```
你是一位物流调度领域的资深分析师。请基于以下图表数据，生成一段凝练的分析解读。

【图表信息】
- 图表名称：{chart_title}
- 图表类型：{chart_type}
- 指标分类：{indicator_type}（Type A 边界类 / Type B 形态类 / Type C 综合效能）
- 对比策略：{strategy_list}
- 数据摘要：{data_summary_json}

【输出要求】
1. 字数控制在 80-150 字
2. 聚焦数据差异的业务含义，不要简单复述数字
3. 使用专业但易懂的语言
4. 如有异常值或显著差异，给出可能的原因分析
5. 如适用，给出简短的优化建议
6. 语气客观中立，避免绝对化表述
```

### 5.3 解读生成流程

报告生成引擎在完成图表数据计算后，将每张图表的数据摘要批量发送给 LLM API。为提高效率，采用并行调用策略：将 16 张图表的解读请求同时发出，使用 `gpt-4.1-mini` 模型以平衡质量与成本。每条解读的生成耗时约 2-3 秒，并行处理后总耗时可控制在 5 秒以内。

### 5.4 解读展示样式

解读文字以带有 AI 图标（🤖）的卡片形式展示在对应图表的正下方，使用略小的字号（13px）和次级文字颜色（#8b95a8），与图表形成视觉层次。卡片左侧带有蓝色竖线装饰，标识其为 AI 生成内容。

## 六、数据流与 API 设计

### 6.1 核心 API 接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 获取订单批次列表 | GET | `/api/projects/{id}/batches` | 返回项目下所有批次及元信息 |
| 获取策略列表 | GET | `/api/projects/{id}/strategies` | 返回已配置的算法策略 |
| 创建报告任务 | POST | `/api/projects/{id}/poc-reports` | 提交生成配置，返回任务 ID |
| 查询报告状态 | GET | `/api/poc-reports/{id}/status` | 轮询生成进度 |
| 获取报告列表 | GET | `/api/projects/{id}/poc-reports` | 返回所有报告记录 |
| 下载报告文件 | GET | `/api/poc-reports/{id}/download/{format}` | 下载 HTML 或 Excel |
| 重新生成报告 | POST | `/api/poc-reports/{id}/regenerate` | 对失败的报告重新触发生成 |

### 6.2 报告生成数据流

```
用户提交配置
    │
    ▼
创建异步任务（状态：生成中）
    │
    ├── 1. 从 C-ROS 仿真引擎获取各策略的仿真结果数据
    │       └── 按批次 × 策略维度拉取车次级别明细数据
    │
    ├── 2. 数据聚合与指标计算
    │       ├── Type A 指标：极值、分位数、均值
    │       ├── Type B 指标：分布统计、KS 检验、CDF
    │       └── Type C 指标：归一化、加权评分
    │
    ├── 3. ECharts 配置生成
    │       └── 将计算结果映射为 16 张图表的 option 配置
    │
    ├── 4. LLM 解读生成（并行）
    │       └── 16 条解读请求 → gpt-4.1-mini → 16 段解读文字
    │
    ├── 5. HTML 报告渲染
    │       └── 将图表配置 + 解读文字注入 HTML 模板 → 单文件输出
    │
    └── 6. Excel 报告生成
            └── 生成多 Sheet 的对比数据表格
    │
    ▼
更新任务状态（状态：已完成 / 失败）
```

### 6.3 HTML 报告文件结构

生成的 HTML 报告为**完全自包含的单文件**，通过 CDN 引用 ECharts 库，所有数据和配置内嵌在 `<script>` 标签中。文件结构如下：

```html
<!DOCTYPE html>
<html>
<head>
    <script src="https://cdn.jsdelivr.net/npm/echarts@5/dist/echarts.min.js"></script>
    <style>/* 内嵌完整样式 */</style>
</head>
<body>
    <!-- 报告元信息 -->
    <!-- KPI 概览卡片 -->
    <!-- Type A/B/C 图表容器 -->
    <!-- 浮动导航 -->
    <script>
        // 全局数据对象
        const REPORT_DATA = { /* 服务端注入 */ };
        // 图表初始化与渲染逻辑
        // LLM 解读文字
    </script>
</body>
</html>
```

## 七、Excel 报告设计

### 7.1 Sheet 结构

在保持与现有 Excel 报告兼容的基础上，增加以下内容：

| Sheet 名称 | 内容 | 新增/保留 |
|-----------|------|---------|

| 批次明细 | 每个批次的详细指标对比 | 保留（扩展多策略列） |
| 车次明细 | 每个车次的详细数据 | 保留 |
| 指标统计 | Type A/B/C 各指标的统计值 | 新增 |
| AI 解读 | 16 张图表的 LLM 解读文字汇总 | 新增 |

### 7.2 多策略列扩展

现有 Excel 的列结构为「人工 | 算法」两列对比。升级后采用动态列结构：

```
指标名称 | 人工经验策略 | 策略A | 策略B | ... | 策略N | 最优标记
```

每个指标行的最后一列标注最优策略名称，便于快速识别。

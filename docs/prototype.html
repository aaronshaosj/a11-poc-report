Type A · 边界类指标对比（极值 / 分位数对齐）
各批次车次数对比
按订单批次 · 分组柱状图
算法在三个批次中的车次数与人工基本持平（±1车次），其中"均衡优化 v2.1"在苏南18批次实现了减少2车次的优化。整体车辆利用效率提升的同时未增加运力投入，符合"车辆数 ≤ 人工"的黄金准则。
各批次平均满载率对比
按订单批次 · 折线图
三组策略的满载率均维持在95%以上的高位水平。"里程优先 v1.3"在苏南17批次满载率略低于人工（-1.6%），这是里程优化策略的合理代价——通过适度降低装载密度换取更短的行驶路径，整体经济性仍优于人工方案。
单车最大里程分布
Max_Distance · 箱线图
算法策略的单车最大里程中位数（85km）低于人工（98km），且上四分位数未超过人工历史极值（135km）。这表明算法在里程约束上表现良好，未出现"超长路线"异常，P95分位数控制在人工Max值以内。
单车最大作业时长分布
Max_Duration · 箱线图
算法策略的作业时长分布呈现明显的双峰特征：短途路线集中在200-350分钟，长途路线在550-750分钟。相比人工调度的均匀分布，算法更倾向于"短线密集配送+长线定向直达"的组合策略，整体时长方差更小。
最大站点间隔对比
按订单批次 · 分组柱状图
最大站点间隔是衡量路线紧凑度的关键指标。算法在17-1批次与人工持平（104.6km），但在苏南17批次出现了较大的站点间隔（+15.8km），提示该批次可能存在跨区配送需求，建议关注OD距离放大系数的设置。
装载量上限利用率（分车型）
Max_weight/volume/qty · 雷达图
雷达图显示算法在重量和体积维度的装载利用率与人工高度一致（差异<2%），但在件数维度上"均衡优化"策略利用率更高（+3.5%），说明算法更擅长在多约束条件下寻找装载平衡点。
Type B · 形态类指标对比（分布形状对齐）
车次订单数分布
Stop Count · 小提琴图
人工调度的订单数集中在5-8单/车次区间（占比62%），呈现明显的右偏分布。算法策略的分布更加均匀，覆盖5-20单/车次的宽幅区间，说明算法能够更灵活地根据站点密度调整每车配送量，避免人工调度中常见的"习惯性固定单量"模式。
车次行驶距离分布
Route_Distance · 密度曲线
行驶距离密度曲线显示，"里程优先"策略的峰值左移至70km处，较人工（85km）缩短约18%。"均衡优化"策略则呈现双峰分布（65km和110km），反映出该策略在短途密集配送和长途跨区配送之间的自适应切换能力。
线路跨度分布（外接圆直径）
Route_Span · 直方图叠加
线路跨度是衡量路线"发散度"的核心指标。算法策略的跨度分布与人工高度吻合（KS检验 p=0.72），均值差异仅0.8km，说明算法生成的路线在地理覆盖范围上与人工经验保持一致，未出现不合理的"放射状"路线。
点间距分布
Stop_Interval · 累积分布函数
CDF曲线显示，90%的算法点间距控制在15km以内，与人工分布高度一致。但算法在5km以内的短间距占比更高（+8%），说明算法更倾向于将地理邻近的站点聚合在同一路线中，提升了配送密度。
跨区数量分布
Cross_Region_Count · 堆叠柱状图
跨区数量直接影响司机的路线认知负担。算法的平均跨区数（1.8区/车次）与人工（2.1区/车次）基本持平，且3区以上的高跨区车次占比更低（-5%），表明OD距离放大系数的设置有效抑制了不必要的跨区行为。
聚类离散度（Top-K 均值）
Nearest_Top_K · 散点对比图
Top-K均值衡量每个站点到同路线内最近K个站点的平均距离，数值越小表示聚类越紧密。算法的Top-3均值（4.2km）优于人工（5.1km），说明算法在站点聚合方面表现更优，路线内部的配送密度更高。
Type C · 综合效能与策略对比
多维效能雷达图
6维指标归一化对比
综合雷达图显示"均衡优化 v2.1"在6个维度中4个维度优于人工（里程、时长、聚类度、点间距），1个维度持平（车次数），仅满载率略低0.3%。"里程优先 v1.3"在里程维度表现最优但牺牲了部分满载率，体现了不同策略的帕累托权衡。
批次维度节降率趋势
Savings Rate · 面积图
里程节降率在三个批次中呈现稳定上升趋势（5.2% → 8.2% → 11.5%），说明算法策略在更大规模的订单集上优化效果更显著。当前最优节降率11.5%已接近"黄金结果"区间（12-18%），建议继续在后续批次中验证收敛性。
车次工作时长 vs 行驶距离（散点矩阵）
双维度联合分布 · 按策略着色
散点矩阵揭示了时长与距离的线性相关性（R²=0.87）。人工调度的散点分布较为集中（距离60-135km，时长210-370min），而算法策略的分布范围更广但回归斜率更低，意味着算法在相同距离下的作业时长更短，路径规划效率更高。

## 五、技术亮点与设计决策

### 与原方案的差异

1. **图表懒加载**: 使用 IntersectionObserver 实现图表延迟渲染，避免 16 张图表同时初始化导致的页面卡顿。图表在即将进入视口（200px 提前量）时才开始渲染。

2. **可复现的 Mock 数据**: 使用带种子的伪随机数生成器（seeded PRNG）+ 高斯分布，确保每次构建生成完全一致的 Mock 数据。每种策略有独立的统计特征配置（profile），真实模拟了不同优化倾向。

3. **简化的组件结构**: 将 StatusBadge、StrategyTag 等小组件内联到 ReportCard 中，减少文件碎片化。StepIndicator、BatchSelector、StrategySelector 内联到 GenerateModal 中，因为它们仅在该上下文使用。

4. **KDE 近似实现**: B2 密度曲线图使用了简化的高斯核密度估计（Gaussian KDE），在前端直接计算，无需额外依赖。

5. **CDF 计算**: B4 累积分布函数图使用经验 CDF 算法，从排序后的车辆明细数据直接计算。

### 技术栈实现

- React 19 + TypeScript (strict mode)
- Tailwind CSS 4（通过 @tailwindcss/vite 插件集成，使用 @theme 自定义属性）
- ECharts 5.6 via echarts-for-react
- wouter 3.9 路由
- clsx 用于条件样式组合
- 无 shadcn/ui 依赖（因无需复杂 UI 组件，直接使用 Tailwind 手写）
